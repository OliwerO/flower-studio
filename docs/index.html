Key decisions made

Price snapshotting on Order Lines

When a florist builds a bouquet, the cost and sell price of each flower is copied into the order line record — not looked up live. This means if a rose costs 2 zł today and 3 zł next month, last month's orders still show the correct margin. Historical KPIs stay accurate forever.

Sequential order creation (no transactions)

Creating an order requires 5–8 separate Airtable API calls (order → lines → stock decrements → delivery). They run one after another. If one fails mid-way, the order is partially created and the owner can fix it in the dashboard. Accepted trade-off: Airtable has no transaction support.

Rate-limit queue (p-queue)

Airtable blocks requests above 5/second. All Airtable calls flow through a queue that enforces this limit automatically. Without it, bulk operations (like order creation) would get HTTP 429 errors.
